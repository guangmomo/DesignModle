1. 状态模式和策略模式的比较
相同点：
1）UML图类似，都符合开闭原则
2）策略模式有初始策略，状态模式有初始状态

不同点：

0)
策略模式封装的是算法
状态模式封装的是状态
1）
策略模式的不同算法之间是相互独立，没有联系的
状态模式的不同状态之间是有联系的，会从一个状态切换到另一个状态
2）
策略模式只能在Context设置策略
状态模式切换状态可以在Context和State中进行


2. 写法：
态模式把对象的行为包装在不同的状态对象里，每一个状态对象都有一个共同的抽象状态基类

3. 优点：
把繁杂的if，else判断简化成一个个状态类，当状态较多的时候，代码更清晰，更容易读懂，更容易扩展
