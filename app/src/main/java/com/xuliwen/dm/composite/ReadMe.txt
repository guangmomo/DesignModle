1. 使用场景：
希望用户忽略组合对象与单个对象的不同，用户将统一地用抽象接口来使用组合结构中的所有对象。

2. 应用场景：
安卓的View（安全的组合模式）
其他树形结构，如文件树

3. 优点：
简化客户端的操作：
因为将“一对多”的关系转化为“一对一”的关系，使得客户代码可以一致的处理对象和对象容器，
无需关心处理的是单个对象，还是组合的对象容器。

4. 写法：
安全的组合模式
透明的组合模式
